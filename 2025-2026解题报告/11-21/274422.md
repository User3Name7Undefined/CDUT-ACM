**摘要：精度问题，双浮点→大整数**

# 总结
C++中包含：
- `log(x)`自然对数
- `log10(x)`以10为底的对数
- `log2(x)`以2为底的对数

计算 $log_{10} x$时，可以直接使用`log10(x)`，而不需要使用`log(x)/log(10)`。

在牛客的评测机上计算 $log_{10} 1000$：

- 以`log(x)/log(10)`计算：得到的答案向下取整为2，可见以此种方法**会产生精度问题**。
- 以`log10(x)`计算：得到的答案向下取整为3，符合预期，但不保证数据变大后不会产生精度问题。

经过进一步测试：
```cpp
cout << log10(999999999999999999) << '\n';
cout << log10(1000000000000000000) << '\n';
```
输出结果为
```cpp
18
18
```

可见，即使使用`log10()`也不可避免地会出现精度问题。

**故，如果位数判断的精度要求高，应当使用比大小/整除10的方法。**

延申可得：

**对于精度要求高的题目，尽量使用大整数而非浮点数。**

例子：[The 2025 ICPC Asia Chengdu Regional Contest - Problem A](https://qoj.ac/contest/2567/problem/14706)。此题vp时卡了很久双精度浮点，最后换成大整数才通过。至今我仍不知道对于此题大整数要开多大以及为什么，vp时只是尽我所能开大然后蒙了一发......
